This demo illustrate how the checkbox default action is done before event propagation, while the reset is done after.
  If you change the type of a checkbox from checkbox to reset during propagation, you will get both default actions run,
  one before and one after the click event.

  Vice versa, you get no default action if you change a type="reset" to type="checkbox" during propagation.

  keydown space on the checkbox will trigger a keypress on the input element, which will then trigger a click.
  preventDefault() only rowks on the keydown, not the keypress
